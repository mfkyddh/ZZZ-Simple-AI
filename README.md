# ZZZ白话讲AI

> 用大白话讲清楚 AI 的基本逻辑。

`ZZZ白话讲AI（ZZZ-Simple-AI）` 是一个面向零基础职场人士和大学生的 AI 入门知识库。

这个项目不追求百科式完整，而是希望用更少的术语、更清楚的结构，帮助读者先建立整体认知，再决定往哪里继续深入。

## 一眼看懂这个项目

| 维度 | 内容 |
| --- | --- |
| 项目目标 | 用白话、短句和结构化内容，把 AI 讲成一套能看懂、能复述的知识系统 |
| 面向人群 | 零基础职场人士、大学生、想先建立整体认知的读者 |
| 内容形态 | Markdown 章节、表格、Mermaid 流程图，后续再生成 HTML |
| 核心方法 | 先问“为什么会出现”，再讲“它是什么”，最后再讲“怎么理解和怎么用” |

## 先抓住四个锚点

这个项目的内容，默认围绕下面四个锚点展开。

| 类型 | 名称 | 你可以先这样理解 |
| --- | --- | --- |
| 两个本质 | 大模型本质上是一个生成文字的概率模型 | 它最底层做的事，是根据前文预测下一个更可能出现的文字片段 |
| 两个本质 | 模型上下文（Context） | 后续很多技术，本质上都在补充、筛选、组织和管理模型当前能看到的信息 |
| 两条主线 | 大模型派（Big Model） | 更关心模型本身能不能继续变强 |
| 两条主线 | 驾驭工程派（Harness Engineering） | 更关心怎样把模型能力接住、管住、组织成稳定系统 |

如果读者先抓住这四个锚点，后面再看检索增强生成（Retrieval-Augmented Generation，`RAG`）、工具调用（Tool Calling）、工作流（Workflow）和智能体（Agent）时，就不容易越看越散。

## 这个项目为什么存在

- 很多 AI 内容太技术化，零基础读者很难进入。
- 很多 AI 内容太零散，知道术语，但不知道主线。
- 这个项目想做的，是先帮读者建立一套能复述、能连接现实场景的理解框架。

## 这个项目怎么讲

- 默认优先使用费曼学习法（Feynman Technique）：先讲到读者能复述，再决定保留多少术语。
- 默认优先使用第一性原理（First Principles）：先回到原始问题，再解释为什么会出现新的技术、产品和框架。
- 例子优先使用职场、校园和日常学习场景，不优先展开底层工程细节。
- 遇到明显流程，优先用 Mermaid 流程图；遇到明显对照，优先用 Markdown 表格。

## 协作约定

- Markdown 是长期内容源，HTML 属于派生产物。
- 术语写法优先以 [项目术语表](./indexes/术语表.md) 为准。
- 新增或重写章节时，优先保证“简单、可复述、贴近真实场景”，不要写成百科式说明。

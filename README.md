# ZZZ白话讲AI

> 把 AI 从“看起来很神秘”讲回“其实可以说清楚”。

`ZZZ白话讲AI（ZZZ-Simple-AI）` 是一个面向零基础职场人士和大学生的开源 AI 入门知识库。

这个项目不追求百科式完整，而是希望用白话、结构化、可复述的方式，解释 AI 到底是什么、为什么会这样发展、以及它在真实工作里到底做了什么。

## 项目是什么

这是一个持续更新的 Markdown 知识库，未来会进一步生成 HTML 网页。

项目的内容会围绕两个底层判断来展开：

- 大模型本质上是一个生成文字的概率模型。
- 后面的很多技术，本质上都在补充、整理和管理模型上下文（Context）。

## 为什么做这个项目

- 很多 AI 内容太技术化，普通读者很难进入。
- 很多 AI 内容太零散，知道术语，却不知道主线。
- 我希望把 AI 讲成一套能读懂、能复述、能持续维护的开源知识系统。

## 面向什么人群

- 零基础职场人士
- 大学生
- 想先理解 AI，再深入技术细节的人

## 这套项目怎么讲 AI

项目当前主要用两条主线来组织内容。

### 1. 底层解释主线

- 大模型本质上是一个生成文字的概率模型。
- 后面的很多技术，本质上都在管理模型上下文（Context）。

### 2. 横向路线主线

- 大模型派（Big Model）
- 驾驭工程 / 外部编排（Harness Engineering）

这里的“驾驭工程 / 外部编排（Harness Engineering）”，你可以先把它理解成：

大模型像一匹很强的马，或者一台动力很强的机器。  
模型本身提供能力，而外部系统通过上下文（Context）、工具、工作流、记忆和约束，把这股能力接住、管住、用起来。

## 项目的特色

- 面向零基础读者，不默认技术背景。
- 强调主线，而不是堆术语。
- 每章都尽量短、稳、结构化，方便后续直接转成网页组件。
- 全项目主要由 Agent 协助编写，人工提供方向、口述想法与校正。

## 内容格式

后续章节默认采用固定 `5` 块模板：

1. `术语`
2. `一句话解释`
3. `你可以先这样理解`
4. `示例`
5. `你只需要记住`

这样做的目的，是让内容更简洁、更稳定，也更方便后续生成网页。

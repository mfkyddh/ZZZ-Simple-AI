# 项目术语表

## 说明

- 这份术语表用于统一本项目的专业术语写法。
- 正文、README、表格、流程图和章节标题，原则上都应优先遵守这里的口径。
- 如果后续发现术语需要修正，应先更新这份术语表，再同步正文。

## 核心术语

| 中文名 | 英文名 | 常用简称 | 推荐写法 | 说明 |
| --- | --- | --- | --- | --- |
| 大语言模型 | Large Language Model | `LLM` | 大语言模型（Large Language Model，`LLM`） | 指生成和处理语言内容的模型 |
| 模型上下文 | Context | — | 模型上下文（Context） | 指模型当前能够看到和利用的信息 |
| 词元 | token | — | 词元（`token`） | 正式中文名；入门阶段可补一句“可以先理解成文字片段” |
| 检索增强生成 | Retrieval-Augmented Generation | `RAG` | 检索增强生成（Retrieval-Augmented Generation，`RAG`） | 指先检索资料，再让模型生成答案 |
| 工具调用 | Tool Calling | — | 工具调用（Tool Calling） | 指模型表达调用工具的意图 |
| 函数调用 | Function Calling | — | 函数调用（Function Calling） | 常见的工具调用实现方式 |
| 模型上下文协议 | Model Context Protocol | `MCP` | 模型上下文协议（Model Context Protocol，`MCP`） | 帮助模型更规范连接工具和数据源的一类协议 |
| 工作流 | Workflow | — | 工作流（Workflow） | 指把多个固定步骤预先串起来的外部流程 |
| 智能体 | Agent | — | 智能体（Agent） | 指围绕目标连续完成多步任务的 AI 应用形态 |
| 技能 | Skill | — | 技能（Skill） | 指可复用的一组说明、工具入口和执行规则 |
| 大模型派 | Big Model | — | 大模型派（Big Model） | 更强调模型本身持续变强的思路 |
| 驾驭工程派 | Harness Engineering | — | 驾驭工程派（Harness Engineering） | 更强调模型外部系统组织能力的思路 |
| 费曼学习法 | Feynman Technique | — | 费曼学习法（Feynman Technique） | 用更简单的话重讲，逼自己确认是否真的理解 |
| 第一性原理 | First Principles | — | 第一性原理（First Principles） | 不从现成说法出发，而是先回到最原始的问题和约束 |

## 当前特别说明

- `token` 在本项目中的正式中文名称统一为 `词元`。
- 如果为了照顾入门读者，可以在第一次出现时补一句：`词元（token）可以先理解成文字片段`。
- 没有常用简称的术语，不写 `简称：无`，直接省略简称字段。

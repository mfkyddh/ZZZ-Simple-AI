## 📖 第一小节：先看这两个产品——生活管家和编程搭档

### ❓ 提问

> 一个帮你处理生活琐事，一个帮你写代码——它们能是一回事吗？

### 🦞 OpenClaw：你的生活管家

想象你有个AI管家，每天早上帮你：
- 查今天的天气和日程
- 给重要的人发提醒
- 整理你下载的文件
- 回复一些简单邮件

你只需要说："帮我安排今天的行程，如果下雨就取消户外活动。"

它会自己查天气、看日历、发消息——**一气呵成，很少停下来问你**。

### 💻 Claude Code：你的编程搭档

再想象你在写代码，有个AI搭档：
- 你说"把这个函数改成异步的"
- 它先给你看它打算怎么改
- 你点头，它才真正动手
- 改完自动跑测试，没过就继续改

每一步都给你看、等你确认——**宁可慢一点，也不能改错**。

### 🤔 表面看，完全不同

| | OpenClaw | Claude Code |
|:---|:---|:---|
| 面对的场景 | 生活琐事 | 编程开发 |
| 工作方式 | 一气呵成 | 步步确认 |
| 出错成本 | 发错邮件能改 | 删错代码难恢复 |
| 给你的感觉 | 像管家，充分授权 | 像搭档，谨慎小心 |

**看起来，这完全是两类产品。**

但等等——第四章说的八部件，它们身上有吗？

## 📖 第二小节：拆开看——原来都是八部件

### 💡 揭晓答案

它们不仅有，而且**一模一样**。

```
        八部件框架（同一套骨架）
        ┌─────────────────────────────────────┐
        │  🧠 核心大脑（大模型做判断）          │
        │  🎯 目标（你要它做什么）              │
        │  📋 上下文（它能看到什么）            │
        │  🧠 记忆（它记得什么）                │
        │  🛠️ 工具（它能调用什么）              │
        │  🔒 边界（什么能做不能做）            │
        │  👁️ 观测（怎么知道做对了）            │
        │  🔄 循环（怎么一步步推进）            │
        └─────────────────────────────────────┘
                 │            │
                 ▼            ▼
           ┌──────────┐  ┌──────────┐
           │ OpenClaw │  │ClaudeCode│
           │ 生活场景 │  │ 代码场景 │
           └──────────┘  └──────────┘
```

---

### 📊 两个产品的八部件

| 部件 | OpenClaw | Claude Code |
|:---:|:---|:---|
| 🧠 **核心大脑** | GPT-5 / Claude 4 | Claude 4.6 |
| 🎯 **目标** | "查天气并提醒朋友" | "把这个函数改成异步" |
| 📋 **上下文** | 天气、日程、联系人 | 代码文件、项目结构 |
| 🧠 **记忆** | 你的习惯、常用邮箱 | 项目配置、构建命令 |
| 🛠️ **工具** | 浏览器、邮件、通讯录 | 文件读写、Bash、测试 |
| 🔒 **边界** | 宽松，充分授权 | 严格，危险操作需确认 |
| 👁️ **观测** | 执行结果直接返回，快速推进 | 测试通过才继续，严格验证 |
| 🔄 **循环** | 快速循环，一气呵成 | 步步确认，谨慎推进 |

---

> 💡 **关键发现**：八部件都有，只是每个部件的"内容"不同——就像同样的剧本，不同的演员演出来完全不同。

> 💡 **补充说明**： OpenClaw 和 Claude Code 都有观测和循环，只是配置不同——OpenClaw 观测简单结果快速推进，Claude Code 通过测试严格验证才继续。这体现了同样的八部件，配置不同，行为完全不同。

**关键发现**：八部件都有，只是每个部件的"内容"不同——就像同样的剧本，不同的演员演出来完全不同。

## 📖 第三小节：学会这套透视法——看透任何 AI 产品

### ❓ 提问

> 搞懂了OpenClaw和Claude Code都是八部件，但这有什么用？  
> 下次遇到通义千问、豆包、DeepSeek，或者某个新出的AI产品，我该怎么快速理解它？

### 📝 术语

- **八部件透视法**：用"核心大脑、目标、上下文、记忆、工具、边界、观测、循环"八个维度分析任何Agent产品的快速方法

### 💬 一句话解释

> 💡 **八部件框架就像一副透视镜——不管AI产品包装得多花哨，用它一看，就知道这产品的"骨架"长什么样。**

### 🧐 你可以先这样理解

前两节我们拆开OpenClaw和Claude Code，发现它们都是八部件。但这只是**开始**——真正有用的是，你能用这套框架去看**任何**AI产品。

就像学会了看房子的"户型图"：
- 不管装修风格是现代还是复古，户型图一看就知道几室几厅
- 八部件就是AI产品的"户型图"

### 📚 八部件透视法怎么用

遇到任何AI产品，问这八个问题：

| 问题 | 你在看什么 | 例子：通义千问 | 例子：豆包 |
|:---|:---|:---|:---|
| **1. 核心大脑是什么？** | 底层用的大模型 | 自研大模型 | 字节大模型 |
| **2. 目标怎么定？** | 它擅长什么类型任务 | 办公写作、文档处理 | 内容创作、聊天 |
| **3. 上下文装什么？** | 能记住多少、记什么 | 多轮对话+上传文档 | 聊天记录为主 |
| **4. 记忆怎么存？** | 长期记忆有没有、存在哪 | 云端同步 | 账号关联 |
| **5. 有什么工具、边界在哪？** | 能调什么能力、限制是什么 | 搜索、画图、代码解释 | 搜索、写作辅助 |
| **6. 观测怎么做？** | 怎么获取执行结果 | 搜索返回结果 | 对话响应 |
| **7. 循环怎么跑？** | 一轮对话还是多轮迭代 | 单轮为主，复杂任务多轮 | 对话式连续 |
| **8. 整体配合怎样？** | 八个部件协同是否顺畅 | 各部件配合成熟 | 侧重对话场景 |

**你不用深入研究每个答案**——只要大概知道这八样东西在哪，这产品的"脾气"你就摸清了。

### 📚 实战演练：快速扫描三个产品

用八部件框架，我们快速"扫描"一下市面上的产品：

**通义千问（阿里）**
- 核心大脑：自研大模型，办公场景优化
- 上下文：支持超长文档、多轮对话
- 工具：搜索、画图、代码解释、PPT生成
- 特点：工具丰富，适合办公场景

**豆包（字节）**
- 核心大脑：字节大模型，内容创作优化
- 上下文：对话连贯性好
- 工具：搜索、写作、图片生成
- 特点：对话自然，适合内容创作

**DeepSeek**
- 核心大脑：推理能力强，代码场景优化
- 上下文：支持长文本、代码理解
- 工具：代码执行、搜索
- 特点：代码和推理任务表现好

**你会发现**：每个产品都在八部件的框架里"做文章"，只是侧重点不同。

### ✅ 你只需要记住

1. **八部件是透视镜**——不管产品包装成什么样，用它一看就懂骨架
2. **问八个问题就够了**——大脑、目标、上下文、记忆、工具、边界、观测、循环
3. **比较产品时看配置**——不是看宣传语，看八部件分别怎么配的

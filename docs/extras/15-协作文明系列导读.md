# 🗺️ 协作文明系列导读

> 这个系列不是教你搭多Agent系统，而是带你建立一种"协作结构感"——让你在看任何AI产品、公司组织架构、甚至菜市场如何定价时，都能识别出背后那套相通的协作逻辑。

---

## 📖 五种协作模式速查

本系列围绕Anthropic提出的五种Multi-Agent协作模式展开。在阅读各篇之前，建议先了解这五种模式的核心定义：

### 1. Generator-Verifier（生成器-验证器）
**一句话理解**：让"写的人"和"审的人"分开，避免自我确认。

**生活化比喻**：就像厨师做菜，不能让厨师自己验收——需要专门的质检员。

### 2. Orchestrator（协调器）
**一句话理解**：一个"信息路由器"负责分解任务、调度执行、汇总结果。

**生活化比喻**：就像装修队的工长，业主只说"我要装修"，工长负责安排水电工、瓦工、木工。

### 3. Agent Teams（固定团队）
**一句话理解**：一组Agent长期协作，形成记忆连续性和协作默契。

**生活化比喻**：就像乐队，成员固定才能配合默契；临时拼凑的乐队很难演出。

### 4. Message Bus（消息总线）
**一句话理解**：发布者发消息不知道谁会收到，订阅者只挑自己关心的——实现解耦。

**生活化比喻**：就像贴公告栏，贴的人不知道谁会看，看的人只挑自己关心的。

### 5. Shared State（共享状态）
**一句话理解**：所有参与者看同一份"事实"，避免"你说的和我说的不是一回事"。

**生活化比喻**：就像共享文档，所有人看的是同一个版本，不是各自手里的不同版本。

---

💡 **你只需要记住**：
- 五种模式是Anthropic总结的Multi-Agent协作基本结构
- 它们不是技术发明，而是人类文明协作结构的显式化
- 本系列将带你理解：为什么文明会"发明"这些结构

## 三个阶段，三种视角

### 第一阶段：模式认知（ext-17 至 ext-21）

逐一拆解Anthropic的五种协作模式：
- 它们是怎么工作的
- 解决什么问题
- 为什么会自然演化出来
- 工程上的边界和局限

**目标**：建立"协作结构感"——看到任何一个复杂系统，能识别出它用了哪种协作结构。

| 编号 | 标题 |
|:---|:---|
| ext-17 | Generator-Verifier：为什么文明一定会发明"审核" |
| ext-18 | Orchestrator：层级制为什么无处不在 |
| ext-19 | Agent Teams：数字员工为什么会出现 |
| ext-20 | Message Bus：为什么市场会战胜中央计划 |
| ext-21 | Shared State：文明真正的核心，其实是共享记忆 |

---

### 第二阶段：文明映射（ext-22 至 ext-25）

从AI模式迁移到人类文明：
- 为什么人类文明一定会演化出这些结构？
- 协作成本、记忆竞争、验证优先——这些底层逻辑如何塑造了我们今天的世界？
- 为什么文明本质上是"协作成本压缩器"？

**目标**：建立"文明结构感"——理解我们身边的组织、制度、市场，本质上都是同一种逻辑的不同表达。

| 编号 | 标题 |
|:---|:---|
| ext-22 | 协作成本：文明真正压缩的到底是什么 |
| ext-23 | 记忆竞争：未来真正的竞争不是智能，而是记忆 |
| ext-24 | 验证优先：为什么未来最稀缺的是"可信协作" |
| ext-25 | 协作演化规律：为什么文明一定会收敛到这些结构 |
